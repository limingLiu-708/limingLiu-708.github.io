git介绍与使用

git介绍

git是开源的分布式版本控制工具。

git官网：https://git-scm.com/

git工作机制

一般工作流程如下：

  1. 克隆 Git 资源作为工作目录
  2. 在克隆的资源上添加或修改文件
  3. 如果其他人修改了，你可以更新资源
  4. 在提交前查看修改
  5. 提交修改
  6. 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交

工作区、暂存区、版本库三者关系

  1. 工作区：就是你在电脑里能看到的目录。
  2. 暂存区：英文叫 stage 或 index。一般存放在 .git 目录下的 index 文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）。
  3. 版本库：工作区有一个隐藏目录 .git，这个不算工作区，而是 Git 的版本库。

git基本操作

  1. workspace：工作区
  2. staging area：暂存区/缓存区
  3. local repository：版本库或本地仓库
  4. remote repository：远程仓库

git常用命令：

命令 作用
git config –global user.name 用户名 设置用户签名
git config –global user.email 邮箱名 设置用户签名
git init 初始化本地库
git status 查看本地库状态
git add 文件名 添加到暂存区
git commit -m “日志信息” 文件名 提交到本地库
git reflog / git log 查看版本历史概要/详细记录
git reset –hard 版本号 版本穿梭，查看历史版本详细信息

Note:

  1. 签名的作用是区分不同操作者身份

git分支管理

版本控制通过分支进行，每一条分支代表一条独立的开发线，使用分支可以从开发主线上分离一个当前版本，用于开发或测试，当前分支的开发调试不影响主线。

git分支实际上是指向更改快照的指针

git分支常用命令

命令 作用
git branch 分支名 创建分支
git checkout 分支名 切换至指定的分支，将工作区的内容替换当前指针最后提交内容
git checkout -b 分支名 切换至指定分支并创建该分支
git merge 分支名 分支合并，以当前所在分支为主分支进行合并
git branch 列出本地分支
git branch -d 分支名 删除分支

git团队协作机制

git远程仓库常用命令

命令 作用
git remote -v 查看当前仓库的所有别名
git remote add 别名 地址/项目名 给当前仓库起别名
git remote rename 旧名 新名 别名换新
git push 仓库地址/别名 分支 将当前本地分支项目推送到远程仓库
git pull 仓库地址/别名 分支 从远程仓库拉取项目到本地分支
git clone 远程仓库地址 克隆指定远程仓库的项目

团队内协作

远程仓库管理者

  1. 创建远程库
  2. 创建本地库并推送至远程库
  3. 邀请协作者加入团队

仓库协作者

  1. 克隆远程仓库到本地
  2. 修改代码并推送至远程仓库

团队间协作

远程仓库管理者

  1. 创建远程库
  2. 创建本地库并推送至远程库
  3. 收到pull request后决定是否merge该分支

仓库协作者

  1. Fork指定远程仓库（为什么是Fork）

  2. 克隆Fork的远程仓库到本地

  3. 修改代码pull request推送分支至远程仓库
